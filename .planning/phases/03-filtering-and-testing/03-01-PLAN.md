  <name>Task 1: Create filter_service.py with individual filter functions</name>
  <files>filter_service.py</files>
  <action>
Create filter_service.py module with four pure functions for filtering schedule data:

1. **filter_by_date(games, date_str)** - Filter games by exact date match
   - Input: list of game dicts, date string in YYYY-MM-DD format
   - Returns: new list containing only games where game['date'] == date_str
   - Handle empty date_str by returning all games (no filter)

2. **filter_by_team(games, team_name)** - Filter games by team (home or away)
   - Input: list of game dicts, team name string
   - Returns: new list containing games where team_name matches either game['homeTeam'] or game['awayTeam']
   - Handle empty team_name by returning all games (no filter)

3. **filter_by_status(games, status)** - Filter games by status
   - Input: list of game dicts, status string ('未开始', '进行中', '已结束')
   - Returns: new list containing games where game['status'] == status
   - Handle empty status by returning all games (no filter)

4. **apply_filters(games, date=None, team=None, status=None)** - Apply multiple filters
   - Input: list of game dicts, optional date/team/status parameters
   - Chain filters: games -> filter_by_date -> filter_by_team -> filter_by_status
   - Returns: new list with all filters applied (intersection of results)
   - Use immutable pattern: each filter creates new list, never mutates input

**Implementation notes:**
- All functions are pure (no side effects, no mutations)
- Use list comprehensions for filtering: `[game for game in games if condition]`
- Empty/None filter values mean "no filter" (return all games for that dimension)
- Document each function with docstring including Args and Returns
- Type hints: `from typing import List, Dict, Optional`
