```python
Handles HTTP fetching and GBK encoding conversion (DATA-01)
"""

import requests
import chardet
from config import SINA_CBA_URL, SINA_USER_AGENT, REQUEST_TIMEOUT, REQUEST_RETRY_COUNT, REQUEST_RETRY_DELAY
from time import sleep
from typing import Optional

def fetch_sina_schedule() -> Optional[str]:
    """
    Fetch CBA schedule HTML from Sina Sports.

    Returns:
        str: Decoded HTML content (UTF-8) if successful
        None: If all retry attempts fail

    Raises:
        RuntimeError: If encoding detection or decoding fails
    """
    # Implementation details:
    # 1. Retry loop with REQUEST_RETRY_COUNT attempts
    # 2. Use requests.get with proper headers (User-Agent from config)
    # 3. Set timeout from config
    # 4. Get response as bytes (response.content, NOT response.text)
    # 5. Try explicit GBK decode first: html = response.content.decode('gbk')
    # 6. If UnicodeDecodeError, fall back to chardet detection
    # 7. Handle network errors (requests.RequestException)
    # 8. Sleep between retries (REQUEST_RETRY_DELAY)
    # 9. Log all attempts and failures to console
```

**Critical GBK handling** (from CONCERNS.md):
- Sina Sports serves GBK-encoded HTML (not UTF-8)
- Use `response.content.decode('gbk')` explicitly (don't rely on requests auto-detection)
- Current Node.js version uses `iconv.decode(buffer, 'gbk')` - Python equivalent is built-in
- Fallback: If explicit GBK fails, use chardet to auto-detect encoding

**Error handling:**
- Wrap requests.get in try/except for RequestException
- Retry with exponential backoff or fixed delay
- Return None if all retries exhausted (don't raise - let caller handle)
- Log detailed error messages

**Testing approach:**

- Print first 500 characters of decoded HTML
