```python
"""
Sina Sports HTML Parser
Extracts game data from HTML tables (DATA-02)
"""

from bs4 import BeautifulSoup
from typing import List, Dict, Optional
from datetime import datetime
import re

def parse_schedule_html(html: str) -> List[Dict]:
    """
    Parse Sina Sports HTML to extract game schedule data.

    Args:
        html: Decoded HTML string from Sina Sports

    Returns:
        List of game dictionaries with keys:
        - round: str (e.g., "第1轮")
        - datetime: str (e.g., "2025-10-20 19:35")
        - home: str (home team name)
        - score: str (e.g., "98:92" or "VS")
        - away: str (away team name)
        - status: str ("未开始", "进行中", "已结束")
    """
    # Implementation approach (from api/schedule.js lines 87-145):
    # 1. Use BeautifulSoup to parse HTML (NOT regex - see CONCERNS.md)
    # 2. Find main table: soup.find('div', class_='part part02')
    # 3. Extract all rows: table.find_all('tr', class_=['odd', 'even'])
    # 4. For each row, extract 5 cells (round, datetime, home, score, away)
    # 5. Determine status based on:
    #    - Has ":" in score and not "VS" → "已结束"
    #    - Game date is today → "进行中"
    #    - Game date is past → "已结束"
    #    - Otherwise → "未开始"
    # 6. Clean text: strip HTML tags, replace &nbsp;, &#58; with ":"

    # Reference: update_schedule.js lines 39-85 (Node.js version)
    # Key difference: Use BeautifulSoup instead of regex for robustness
```

**Edge case handling** (from CONCERNS.md):
- Missing data: If cell is empty, use sensible defaults ("待定" for team, "VS" for score)
- Date parsing: Handle various formats ("2025-10-20 19:35", "2025-10-20", etc.)
- Status logic: Exact match current implementation (lines 64-74 in update_schedule.js)
- HTML entities: BeautifulSoup handles automatically, but verify `:` in scores

**Data cleaning:**
- Strip all HTML tags within cells: `cell.get_text(strip=True)`
- Replace `&nbsp;` with space (BeautifulSoup does this)
- Normalize whitespace: `re.sub(r'\s+', ' ', text).strip()`

**Why BeautifulSoup over regex** (from CONCERNS.md line 139):
- Current implementation uses fragile regex that breaks when Sina changes markup
- BeautifulSoup uses CSS selectors and handles malformed HTML
- More maintainable and less likely to break

```python
import json
from pathlib import Path

def load_schedule_data():
    """Load schedule from embedded JSON file"""
    schedule_path = Path(__file__).parent / 'data' / 'schedule.json'
    try:
        with open(schedule_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        return data['schedule']
    except FileNotFoundError:
        print(f"Warning: schedule.json not found at {schedule_path}")
        return []
    except json.JSONDecodeError as e:
        print(f"Error parsing schedule.json: {e}")
        return []

# Load schedule on module import (cached)
SCHEDULE_DATA = load_schedule_data()
```

2. **Update index() route** (replace sample_schedule):
```python
@app.route('/')
def index():
```

**Why load at module level:**

**Error handling:**
- FileNotFoundError: Log warning but don't crash (allows development without data)
- JSONDecodeError: Log error and return empty list (malformed JSON)
