```python
@app.route('/api/schedule')
def api_schedule():
    """
    API endpoint returning CBA schedule as JSON (DATA-04)

    Response format:
    {
        "success": true,
        "data": [...schedule array...],
        "count": 100,
        "updated": "2026-03-23T10:30:00Z"
    }
    """
    try:
        # Read schedule.json to get updated timestamp
        schedule_path = Path(__file__).parent / 'data' / 'schedule.json'
        with open(schedule_path, 'r', encoding='utf-8') as f:
            full_data = json.load(f)

        return jsonify({
            'success': True,
            'data': SCHEDULE_DATA,
            'count': len(SCHEDULE_DATA),
            'updated': full_data.get('updated', datetime.utcnow().isoformat() + 'Z')
        })
    except Exception as e:
        # Error response
        return jsonify({
            'success': False,
            'error': str(e),
            'data': []
        }), 500
```

**Response format** (matches api/schedule.js lines 32-36):
- `success`: Boolean indicating if request succeeded
- `data`: Array of game objects (same structure as SCHEDULE_DATA)
- `count`: Number of games in data array
- `updated`: ISO 8601 timestamp of last data update

**Error handling:**
- Catch any exception (file read errors, JSON parsing)
- Return 500 status code with error details
- Include empty data array in error response (client compatibility)

**CORS headers:**
- Already handled by @app.after_request decorator (lines 43-49 in current app.py)
- No additional CORS setup needed

**Performance:**
- Uses cached SCHEDULE_DATA (no file read on every request)
- Only reads schedule.json to get 'updated' timestamp (small overhead)
- Could optimize further by caching timestamp too, but not critical for v1
